## Team Communication Protocol

### Daily Standup
- **Time**: Start of development day
- **Format**: Async updates in team channel
- **Contents**: Progress, blockers, next steps

### Weekly Sync
- **Time**: End of each week
- **Duration**: 1 hour
- **Participants**: All agents + human oversight
- **Topics**: Phase progress, metrics, risks

### Task Delegation
- **Format**: Structured task with clear requirements
- **Validation**: Agent confirms understanding before starting
- **Handoff**: Completed work with test results
- **Review**: Code review before merge

## Key Principles

1. **Architecture First**: All code must match architecture.md
2. **Token Efficiency**: Always use jcodemunch MCP for code retrieval
3. **Quality Over Speed**: 80%+ test coverage mandatory
4. **Clear Communication**: Report progress and blockers immediately
5. **Human Oversight**: Critical decisions require approval
6. **Security First**: No secrets in code, security review mandatory
7. **Documentation**: Document as you code
8. **Continuous Integration**: Commit locally, test thoroughly
